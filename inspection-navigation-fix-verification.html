点巡检管理导航修复验证

修复说明

问题：点巡检管理页面有两个左侧导航，结构不正确。

解决方案：将巡检路线、巡检计划、巡检任务、巡检报告作为三级菜单放在主导航的"设备管理 > 点巡检管理"下。

修改内容

1. index.html - 导航配置

在设备管理的二级菜单中，将"点巡检管理"配置为带有三级菜单的项：

  • 巡检路线 → pages/equipment/inspection-management.html?view=routes
  • 巡检计划 → pages/equipment/inspection-management.html?view=plans
  • 巡检任务 → pages/equipment/inspection-management.html?view=tasks
  • 巡检报告 → pages/equipment/inspection-management.html?view=reports

2. inspection-management.html - 移除内部导航

移除了页面内部的左侧导航栏（109-142行），现在使用主导航系统。

3. equipment-inspection.js - URL参数支持

更新JavaScript逻辑：

  • 根据URL参数（?view=routes/plans/tasks/reports）自动切换视图
  • 移除对内部导航按钮的引用
  • 添加initializeViewDisplay()函数初始化视图显示状态

验证步骤

  1. 打开主页面（index.html）
  2. 点击左侧导航"设备管理"
  3. 在二级菜单中找到"点巡检管理"
  4. 点击"点巡检管理"，应该展开显示四个三级菜单项：
    • 巡检路线
    • 巡检计划
    • 巡检任务
    • 巡检报告
  5. 点击任意三级菜单项，应该加载对应的视图
  6. 确认页面中只有一个左侧导航（主导航），没有页面内部的第二个导航

快速测试链接

打开主页面 巡检路线 巡检计划 巡检任务 巡检报告

预期结果

主导航中"设备管理"下的"点巡检管理"可以展开显示四个子菜单

点击子菜单项后，页面只显示一个左侧导航（主导航）

内容区域根据选择的子菜单显示对应的视图（路线/计划/任务/报告）

页面标题和内容与选择的菜单项匹配
