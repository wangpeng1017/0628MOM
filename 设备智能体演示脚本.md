### 场景1：智能引导式故障诊断（推荐首选）⭐

**演示重点：**
- 自然语言交互
- 多轮对话能力
- 知识推荐（相似案例、SOP文档）
- 智能诊断引导
- 知识自动沉淀

**演示步骤：**

1. **点击"场景1"按钮**
2. **第一轮对话：**
   - 用户输入：`这台绕线机老是断线，怎么回事？`
   - AI回复：询问具体发生阶段（启动/高速运行/停止）

3. **第二轮对话：**
   - 用户输入：`主要是在高速运行时`

     - 相关SOP文档
     - 诊断引导建议
   - **讲解点：** RAG检索增强，从知识库找到最相关案例

4. **第三轮对话：**
   - 用户输入：`张力值不稳定，在50-100g之间跳动`
   - AI回复：定位根因 + 技术文档 + 推荐备件
   - **讲解点：** 
     - 精准定位问题（张力传感器故障）
     - 自动调取技术图纸
     - 实时显示备件库存

5. **第四轮对话：**
   - 用户输入：`发现传感器信号线绝缘皮有磨损，已重新处理，张力恢复稳定`
   - AI回复：维修记录 + 知识沉淀
   - **讲解点：** 
     - 自动记录维修过程

## 📝 演示后的FAQ准备

### 技术实现类问题：

**Q: 使用的是什么大模型？**
A: 我们支持多种模型，包括开源的ChatGLM、Qwen，也可以对接GPT-4等商业API。会根据客户需求和预算推荐最适合的方案。

**Q: RAG是什么技术？**
A: RAG是检索增强生成技术，简单说就是先从知识库找到相关资料，再让AI基于这些资料回答，保证了答案的准确性和专业性。

**Q: 知识库怎么建立？**
A: 初期导入现有的维修记录、SOP文档，后期系统会自动从每次维修中学习和积累。

### 业务价值类问题：

**Q: ROI如何计算？**
A: 主要体现在三个方面：
1. 降低维修时间（平均节省30-50%）
2. 减少重复故障（通过知识沉淀）
3. 加快新人培养（3个月变1个月）

**Q: 实施周期多长？**
A: 分阶段实施：
- Phase 1（1个月）：基础问答和知识检索
- Phase 2（2个月）：智能诊断和系统集成
- Phase 3（持续）：优化和迭代
